I'm sure that you must be asking yourself this question right now: why do we skip self?

Let's see why...

We do this because the value of self is assigned automatically by the program behind the scenes.

For example:

>>> class House:
	
	def __init__(self, price):
		self.price = price

>>> my_house = House(14000)

We didn't pass a value for self explicitly, but the code worked correctly, so let's check if self has a value.

If we add a print statement to print the value of self in __init__():

>>> class House:
	
	def __init__(self, price):
		print(self) # New print statement
		self.price = price

>>> my_house = House(14000)
<__main__.House object at 0x032AE0B8> # Output

We can confirm that self has a value even though we didn't assign it explicit when we created the instance.

This value was assigned automatically to reference the new object.
